#  Food Delivery Business Insights using SQL

##  Project Overview

This project analyzes a Food Delivery database containing Customers, Orders, and Restaurants data. The objective was to solve real-world business problems using SQL and generate meaningful insights into customer behavior, restaurant performance, revenue trends, and payment preferences.


##  Database Schema

The database consists of three tables:

### Customers

* Customer ID
* Customer Name
* City

### Orders

* Order ID
* Customer ID
* Restaurant ID
* Order Amount
* Order Date
* Payment Method
* Order Status

### Restaurants

* Restaurant ID
* Restaurant Name

##  SQL Concepts Used

* INNER JOIN
* LEFT JOIN
* Aggregate Functions
* GROUP BY
* ORDER BY
* CASE WHEN
* Common Table Expressions (CTEs)
* Subqueries
* Window Functions

  * ROW_NUMBER()
  * DENSE_RANK()
  * LAG()



##  Business Problems Solved

1. Display completed orders along with customer and restaurant details.
2. Calculate total revenue generated by each restaurant.
3. Find total orders placed by each customer.
4. Analyze city-wise order values.
5. Categorize orders as Low, Medium, and High value.
6. Compare restaurant performance against overall averages.
7. Identify customers who never placed orders.
8. Find top spending customers.
9. Determine the most frequently used payment method.
10. Calculate running revenue totals.
11. Rank restaurants based on revenue.
12. Find the second highest order amount.
13. Identify customers with above-average order frequency.
14. Find top restaurants in each city.
15. Analyze month-over-month revenue trends.

##  Key Insights

* Identified top-performing restaurants by revenue.
* Analyzed customer spending patterns.
* Evaluated payment method preferences.
* Ranked restaurants using window functions.
* Tracked revenue performance across months.
* Identified inactive customers for potential marketing campaigns.
